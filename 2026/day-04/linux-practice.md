At the end of **Day 04 of the #90DaysOfDevOps challenge**, I practiced essential Linux commands that are commonly used for process monitoring, service management, and log analysis.

---

## Practice Overview

### Process Inspection
- Checked running processes and system load using `ps aux` and `top`
- Observed CPU and memory usage of active processes

### Service Management
- Inspected a systemd-managed service using `systemctl status`
- Reviewed active services using systemd tools

### Log Analysis
- Examined service-specific logs using `journalctl`
- Reviewed recent system logs using `tail`

This exercise helped reinforce how **processes, services, and logs are interconnected** when diagnosing issues on a Linux system.
